Project name:

Core concept:
The app allows employees to book hot desks and meeting rooms from an interactive office floor plan. Users can select a date, time, space type, capacity, and required equipment. The system recommends the best available desk or room based on availability, capacity, equipment, location, and booking conflicts.

Recommendation logic:
Create a backend service that recommends the best available spaces based on:
1. Space must be active.
2. Space must be available during the selected time.
3. Space type must match the requested type.
4. Capacity must be equal to or greater than attendee_count.
5. Space must include all required equipment.
6. Prefer the smallest suitable capacity.
7. Prefer spaces closest to the user’s preferred zone if provided.
8. Prefer spaces with the most matching optional equipment.
9. Return a ranked list with score and explanation.

Conflict handling:
Prevent double booking using this overlap condition:
existing.start_time < requested_end_time AND existing.end_time > requested_start_time

Implement transaction-safe booking creation to reduce race condition risk. Use database constraints, transaction isolation, or row-level locking where possible. Add clear comments explaining how this protects against double booking.

Security requirements:
Implement protections for:

1. SQL Injection
- Use SQLAlchemy ORM or parameterised queries only.
- Never concatenate user input into raw SQL.

2. XSS
- React should escape rendered text by default.
- Avoid dangerouslySetInnerHTML.
- Validate and sanitise user-generated fields.

3. CSRF / Auth Security
- If using JWT, store tokens securely and avoid unsafe localStorage patterns where possible.
- If using cookies, use HttpOnly, Secure, SameSite cookies.
- Protect authenticated API routes.

4. Broken Access Control
- Users can only edit/cancel their own bookings.
- Admin-only endpoints must require admin role.
- Prevent IDOR vulnerabilities.

5. Input Validation
- Use Pydantic schemas.
- Validate start/end times.
- Prevent bookings in the past.
- Prevent negative or unrealistic attendee counts.
- Validate equipment IDs and space IDs.

6. Audit Logging
- Log booking creation, cancellation, admin changes, and failed permission attempts.
